## 1. Физические основы

### 1.1 Проблема: упругость филамента

В FDM-печати экструдер представляет собой упругую механическую систему. Филамент (пластиковая нить) сжимается под давлением в хотэнде. Когда головка начинает движение с ускорением, давление в сопле падает — возникает **недоэкструзия** (under-extrusion). При завершении движения давление возрастает — возникает **переэкструзия** (over-extrusion). Этот эффект проявляется в виде дефектов на углах и резких переходах.

```
Скорость печати:          ───────╱╲───────
                                   ╲╱
Давление в сопле:               ╱──────╲
(с задержкой)              ╱──────────────╲

Расход пластика:           ╱──╲    ╱──╲
(с выбросом)              ╱    ╲──╱    ╲
                           недолив  перелив
```

### 1.2 Идея Pressure Advance

**Pressure Advance (PA)** — это метод компенсации, при котором экструдер опережает (или отстаёт от) движение печатающей головки, чтобы поддерживать постоянное давление в сопле.

Классический линейный advance (Marlin LinAdvance) использует модель:

```
E_коррекция = K × V_печати × ускорение
```

Это требует знания ускорения, производной скорости и даёт сильную связь между параметрами движения.

### 1.3 Упрощённый PA: дифференциальная модель (SPA v4.4)

**SPA (Simplified Pressure Advance)** реализует **дифференциальную модель**, основанную на изменении скорости экструзии между последовательными временными отсчётами (кадрами FT Motion):

```
ΔVe = Ve[текущий] - Ve[предыдущий]
E_компенсация += K × ΔVe
```

**Ключевые отличия от классического LinAdvance:**
- Не использует EMA-фильтр (τ / постоянная времени удалены)
- Не требует знания ускорения блока
- Работает в фиксированном временном кадре `FTM_TS = 1 / FTM_FS`
- Компенсация накапливается (интегратор), а не вычисляется заново каждый кадр
- Минимальная задержка — один кадр (≈ 100–200 мкс)

## 2. Математическая модель

### 2.1 Обозначения

| Символ | Описание | Единицы |
|--------|----------|---------|
| `K` | Коэффициент Pressure Advance | с (секунды) |
| `Ve[t]` | Скорость экструзии в момент t | мм/с |
| `E_planned[t]` | Плановое положение экструдера | мм |
| `E_corrected[t]` | Скорректированное положение экструдера | мм |
| `ΔVe[t]` | Изменение скорости экструзии | мм/с |
| `offset[t]` | Накопленная компенсация | мм |
| `FTM_FS` | Частота дискретизации FT Motion | Гц |
| `FTM_TS` | Период кадра = 1 / FTM_FS | с |
| `Q16` | Формат фиксированной точки: 16 дробных битов | — |

### 2.2 Уравнения

**Шаг 1. Вычисление текущей скорости экструзии:**

```
Ve[t] = (E_planned[t] - E_planned[t-1]) × FTM_FS      (1)
```

где `FTM_FS = 1 / FTM_TS` — количество кадров в секунду.

**Шаг 2. Вычисление изменения скорости:**

```
ΔVe[t] = Ve[t] - Ve[t-1]                                (2)
```

**Шаг 3. Вычисление приращения компенсации:**

```
Δoffset[t] = K × ΔVe[t]                                  (3)
```

**Шаг 4. Накопление компенсации (только при экструзии):**

```
offset[t] = offset[t-1] + Δoffset[t], если pa_extruding  (4)
```

**Шаг 5. Скорректированное положение экструдера:**

```
E_corrected[t] = E_planned[t] + offset[t]                (5)
```

### 2.3 Анализ размерности

Проверим размерность коэффициента `K`:

```
Δoffset [мм] = K [???] × ΔVe [мм/с]
```

Для корректной размерности: `K` должен иметь размерность **секунды [с]**.

```
Δoffset [мм] = K [с] × ΔVe [мм/с] = [мм] ✓
```

Таким образом, `K = 0.1` означает, что при изменении скорости экструзии на 10 мм/с компенсация составит 1 мм.

### 2.4 Q16 — формат фиксированной точки

Все вычисления SPA выполняются в целочисленной арифметике **Q16** (16 дробных битов) для максимальной производительности на микроконтроллерах без FPU.

```
Значение_Q16 = round(значение_float × 2^16) = round(значение_float × 65536)

float_value = Q16_value / 65536.0
```

**Используемые переменные в Q16:**

| Переменная | Описание | Q16 |
|-----------|----------|-----|
| `ftmotion_pa_k_q16` | K × 65536 | int32_t |
| `spa_ve_prev_q16` | Ve[t-1] × 65536 | int32_t |
| `spa_pa_offset_q16` | offset[t] × 65536 | int32_t |

### 2.5 Преобразование уравнений в Q16

**Скорость экструзии в Q16:**

```
Ve_Q16[t] = (E_planned[t] - E_planned[t-1]) × 65536 × FTM_FS    (1Q)
```

**Изменение скорости:**

```
dVe_Q16 = Ve_Q16[t] - Ve_Q16[t-1]                                 (2Q)
```

**Приращение компенсации (64-битное умножение для предотвращения переполнения):**

```
Δoffset_Q16 = (K_Q16 × dVe_Q16) >> 16                             (3Q)
```

> Примечание: умножение двух int32_t даёт int64_t, затем сдвиг на 16 бит возвращает обратно в Q16.

### 2.6 Поведение интегратора

SPA использует **накапливающийся** (интегрирующий) offset, а не мгновенную компенсацию. Это даёт:

- **Плавность**: компенсация не дёргается при кратковременных флуктуациях
- **Автоматический возврат к нулю**: после завершения движения ΔVe = 0 → offset перестаёт меняться
- **Устойчивость**: интегратор не накапливает ошибку, так как ΔVe — знакопеременная величина

Однако, offset может стать значительным при длительном ускорении. При `K = 0.2` и разгоне с 0 до 100 мм/с за 0.1 с:

```
a = 1000 мм/с²
ΔVe_per_frame = a × FTM_TS ≈ 1000 × 0.0002 = 0.2 мм/с (при FTM_FS = 5000 Гц)

За время разгона (0.1 с) будет 500 кадров.
total_offset = K × ΣΔVe = 0.2 × 100 = 20 мм
```

Это ожидаемое поведение — экструдер должен "убежать вперёд" на 20 мм для создания необходимого давления.

## 3. Программная реализация

### 3.1 Архитектура

SPA интегрирован в систему FT Motion (Fixed-Time Motion). Общая архитектура:

```
┌─────────────────────────────────────────────────────────────────────────┐
│                        FTMotion::loop()                                 │
│                                                                         │
│  ┌──────────┐    ┌──────────────────────┐    ┌───────────────────┐     │
│  │planner   │───→│fill_stepper_plan_    │───→│stepping_t (buffer)│     │
│  │(block    │    │buffer()              │    │                   │     │
│  │ queue)   │    │                      │    │ step_pulse gen.   │     │
│  └──────────┘    │  for each frame:     │    └─────────┬─────────┘     │
│                  │    calc_traj_point() │              │                │
│                  │    stepping_enqueue()│              ▼                │
│                  └──────────────────────┘        Stepper ISR           │
│                                                                         │
│                  calc_traj_point() содержит SPA                         │
└─────────────────────────────────────────────────────────────────────────┘
```

### 3.2 Глобальные переменные

Определены в [`Marlin/src/module/ft_motion.cpp`](Marlin/src/module/ft_motion.cpp:54):

```cpp
#if ENABLED(SIMPLIFIED_PA)
// === SPA v4.4: Дифференциальная модель (K·ΔVe), без EMA и τ ===
int32_t ftmotion_pa_k_q16 = 0;                  // Коэффициент K в формате Q16 (K_float × 65536)
static int32_t spa_ve_prev_q16 = 0;             // Предыдущая скорость экструзии (Q16), мм/с × 65536
static int32_t spa_pa_offset_q16 = 0;           // Накопленная компенсация PA (Q16), мм × 65536
```

### 3.3 Основные функции

#### 3.3.1 Установка коэффициента `K` — [`ftmotion_pa_set_k()`](Marlin/src/module/ft_motion.cpp:61)

```cpp
void ftmotion_pa_set_k(float k_new) {
  ftmotion_pa_k_q16 = int32_t(k_new * 65536.0f);
}
```

- Преобразует float K в Q16 (умножение на 65536)
- Вызывается из [`M900`](Marlin/src/gcode/feature/advance/M900.cpp:122) с параметром `K`
- Диапазон: 0.0–2.0 с

#### 3.3.2 Сброс состояния — [`ftmotion_pa_reset_state()`](Marlin/src/module/ft_motion.cpp:66)

```cpp
void ftmotion_pa_reset_state() {
  spa_ve_prev_q16 = 0;
  spa_pa_offset_q16 = 0;
}
```

- Обнуляет предыдущую скорость и накопленную компенсацию
- Вызывается при:
  - [`G92 E0`](Marlin/src/gcode/geometry/G92.cpp:134) — сброс позиции экструдера
  - [`M600` / замена филамента](Marlin/src/feature/pause.cpp:399) — пауза
  - Обрыв филамента (runout)

После сброса требуется ~1–2 кадра для восстановления компенсации, что допустимо.

#### 3.3.3 Чтение `K` — [`ftmotion_pa_get_k()`](Marlin/src/module/ft_motion.h:437)

```cpp
inline float ftmotion_pa_get_k() { return ftmotion_pa_k_q16 / 65536.0f; }
```

- Инлайн-функция, преобразует Q16 обратно в float

#### 3.3.4 Основной алгоритм — [`calc_traj_point()`](Marlin/src/module/ft_motion.cpp:518)

Алгоритм встроен в функцию расчета траекторной точки:

```cpp
// Внутри FTMotion::calc_traj_point(), секция SIMPLIFIED_PA:

#if ENABLED(SIMPLIFIED_PA)
  static block_t* last_block = nullptr;
  block_t* current_block = stepper.get_current_block();

  if (current_block != last_block) {
    last_block = current_block;
  }

  if (current_block && current_block->pa_active) {
    // (1) Текущая скорость экструзии [Ve = ΔE / Δt]
    const float e_planned = traj_coords.e;
    const int32_t ve_curr_q16 = int32_t((e_planned - prev_traj_e) * 65536.0f * FTM_FS);

    // (2) Изменение скорости [dVe = Ve_curr - Ve_prev]
    const int32_t dve_q16 = ve_curr_q16 - spa_ve_prev_q16;

    // (3) Приращение компенсации [Δoffset = K × dVe]
    const int64_t pa_delta_q16 = ((int64_t)ftmotion_pa_k_q16 * (int64_t)dve_q16) >> 16;

    // (4) Накопление offset (только при экструзии)
    if (current_block->pa_extruding) {
      spa_pa_offset_q16 += (int32_t)pa_delta_q16;
    }

    // (5) Обновление предыдущей скорости и положения
    spa_ve_prev_q16 = ve_curr_q16;
    prev_traj_e = e_planned;

    // (6) Применение компенсации
    traj_coords.e += (float)spa_pa_offset_q16 * (1.0f / 65536.0f);
  }
#endif
```

### 3.4 Вспомогательные структуры данных

Определены в [`Marlin/src/module/planner.h:322-326`](Marlin/src/module/planner.h):

```cpp
#if ENABLED(PA_LOOKAHEAD)
  int32_t pa_K_q16;           // K для этого блока (копия на момент запуска)
  bool pa_active;             // Флаг активности SPA для блока
  bool pa_extruding;          // Флаг: есть шаги экструдера (steps.e > 0)
#endif
```

- `pa_K_q16` — фиксация `K` на момент постановки блока в очередь. Защита от смены `K` на лету.
- `pa_active` — сигнал для `calc_traj_point()` применять SPA
- `pa_extruding` — блокирует накопление offset при ретрактах (движение филамента назад)

### 3.6 Сброс очереди при G92

В [`Marlin/src/module/planner.h:1167`](Marlin/src/module/planner.h):

```cpp
#if ENABLED(PA_LOOKAHEAD)
  static void pa_flush_queue() {
    for (uint8_t i = block_buffer_tail; i != block_buffer_head;
         i = (i + 1) % BLOCK_BUFFER_SIZE) {
      block_buffer[i].pa_active = false;
    }
  }
#endif
```

Деактивирует SPA для всех блоков в очереди после `G92 E0`, чтобы не было скачка offset при сбросе позиции экструдера.

## 5. Сценарии использования

### 5.1 Настройка K

Рекомендуемый метод настройки:

1. Напечатать тестовую модель (PA-башню) с изменением K от 0.00 до 0.30
2. Визуально оценить качество углов:
   - **K слишком мал**: недоэкструзия на углах (провалы)
   - **K слишком велик**: переэкструзия на углах (наплывы/пузыри)
   - **K оптимален**: ровные, чёткие углы
3. Установить значение в `M900 K<value>` или в конфигурации

Типичные значения: 0.05–0.30 в зависимости от:
- Вязкости пластика (PLA ниже, PETG выше)
- Длины хотэнда
- Скорости печати

### 5.2 Смена филамента

При `M600` или обрыве филамента:
1. Вызывается [`ftmotion_pa_reset_state()`](Marlin/src/feature/pause.cpp:400)
2. Сбрасывается накопленный offset и предыдущая скорость
3. После замены SPA восстанавливается за 1–2 кадра

### 5.3 Сброс позиции экструдера (G92 E0)

При `G92 E0`:
1. Вызывается [`ftmotion_pa_reset_state()`](Marlin/src/gcode/geometry/G92.cpp:136)
2. Вызывается [`pa_flush_queue()`](Marlin/src/gcode/geometry/G92.cpp:137) — деактивирует SPA для всех блоков в очереди
3. Это предотвращает скачок offset при обнулении координат экструдера

### 5.4 Совместимость с другими системами

| Компонент | Совместимость | Примечание |
|-----------|--------------|------------|
| FT Motion | Обязателен | SPA работает только в связке с FT Motion |
| Input Shaping | Полная | SPA применяется до shaping/smoothing |
| Smoothing | Полная | Не влияет на работу сглаживания |
| M900 | Да | Используется для установки K |
| G92 E0 | Да | Сброс состояния |
